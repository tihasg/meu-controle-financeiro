# Expense Manager - Code Snippets & Examples

## 🔨 Key Code Patterns

### 1. Database Initialization

```kotlin
// In main.kt
fun main() = application {
    val database = ExpenseDatabase()
    
    Window(
        onCloseRequest = {
            database.close()
            exitApplication()
        },
        title = "Gerenciador de Despesas"
    ) {
        ExpenseManagerUI(database)
    }
}
```

**What happens**:
- ExpenseDatabase() is called
- init block runs: Class.forName("org.sqlite.JDBC")
- initializeDatabase() creates table if not exists
- Window opens with ExpenseManagerUI
- On close, database connection is properly closed

---

### 2. Adding an Expense

```kotlin
// Create new expense
val newExpense = Expense(
    fornecedor = "Telefônica",
    categoria = "Internet",
    valor = 120.50,
    status = "Aguardando",
    vencimento = LocalDate.of(2026, 3, 31)
)

// Add to database
val id = database.addExpense(newExpense)

// Refresh UI
expenses = database.getAllExpenses()
```

**Note**: ID is auto-generated by SQLite

---

### 3. Toggling Status

```kotlin
// Toggle between statuses
val newStatus = if (expense.status == "Pago") "Aguardando" else "Pago"

// Update in database
database.updateExpenseStatus(expense.id, newStatus)

// Refresh list
expenses = database.getAllExpenses()
```

**Colors**:
- Green button: "Pago"
- Orange button: "Aguardando"

---

### 4. Deleting an Expense

```kotlin
// Delete by ID
database.deleteExpense(expense.id)

// Refresh list to remove from UI
expenses = database.getAllExpenses()
```

**Warning**: This action cannot be undone

### 6. State Management

```kotlin
// Store expenses list in composable state
var expenses by remember { mutableStateOf(database.getAllExpenses()) }

// Update state triggers recomposition
expenses = database.getAllExpenses()

// UI automatically updates when state changes
LazyColumn {
    items(expenses) { expense ->
        ExpenseItem(expense = expense, ...)
    }
}
```

### 9. SQL Query Examples

**SELECT all expenses sorted by amount descending**:
```sql
SELECT * FROM expenses ORDER BY valor DESC
```

**INSERT new expense**:
```sql
INSERT INTO expenses (fornecedor, categoria, valor, status, vencimento)
VALUES ('Telefônica', 'Internet', 120.50, 'Aguardando', '2026-03-31')
```

**UPDATE expense status**:
```sql
UPDATE expenses SET status = 'Pago' WHERE id = 1
```

**DELETE expense**:
```sql
DELETE FROM expenses WHERE id = 1
```

---

### 10. Parameterized Queries (Safe from SQL Injection)

```kotlin
// Safe: Using prepared statement with ?
val stmt = conn.prepareStatement(
    "UPDATE expenses SET status = ? WHERE id = ?"
)
stmt.setString(1, newStatus)
stmt.setInt(2, expenseId)
stmt.executeUpdate()

// This prevents SQL injection attacks
// Always use this pattern instead of string concatenation
```

## 📈 Performance Considerations

### Database Queries
- Queries run on main thread (acceptable for small datasets)
- For large datasets, consider using coroutines
- Indexes could be added on frequently queried columns

### UI Rendering
- LazyColumn renders only visible items (efficient)
- remember { } prevents unnecessary recompositions
- No memory leaks with proper resource cleanup

### Memory Usage
- Database keeps single persistent connection
- UI state is minimal (just a list and boolean)
- No memory waste from unfinished transactions
